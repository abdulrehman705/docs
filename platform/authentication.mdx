Every request to the Buildmarkets API must include your credentials in the `Authorization` header using HTTP Basic Authentication. Your **API Key** acts as the username and your **API Secret** acts as the password — you receive a separate key pair for sandbox and production.

## How It Works

HTTP Basic Auth encodes your credentials as a Base64 string and sends them with every request. Follow these three steps to build the header value:

<Steps>
  <Step title="Concatenate your credentials">
    Join your API Key and API Secret with a colon separator:

    ```text
    API_KEY:API_SECRET
    ```
  </Step>
  <Step title="Base64-encode the result">
    Encode the concatenated string using standard Base64:

    ```bash
    echo -n 'YOUR_API_KEY:YOUR_API_SECRET' | base64
    ```
  </Step>
  <Step title="Send the Authorization header">
    Prefix the encoded value with `Basic ` and include it in every request:

    ```text
    Authorization: Basic <base64_encoded_value>
    ```
  </Step>
</Steps>

## Code Examples

<CodeGroup>

```bash cURL
curl -X GET 'https://test-service.bluumfinance.com/v1/assets' \
  -H 'Authorization: Basic '$(echo -n 'YOUR_API_KEY:YOUR_API_SECRET' | base64)
```

```javascript Node.js
const axios = require('axios');

const API_KEY = process.env.BLUUM_API_KEY;
const API_SECRET = process.env.BLUUM_API_SECRET;

const credentials = Buffer.from(`${API_KEY}:${API_SECRET}`).toString('base64');

const response = await axios.get('https://test-service.bluumfinance.com/v1/assets', {
  headers: {
    Authorization: `Basic ${credentials}`,
    'Content-Type': 'application/json',
  },
});

console.log(response.data);
```

```python Python
import base64
import os
import requests

api_key = os.environ['BLUUM_API_KEY']
api_secret = os.environ['BLUUM_API_SECRET']

credentials = base64.b64encode(f'{api_key}:{api_secret}'.encode()).decode()

response = requests.get(
    'https://test-service.bluumfinance.com/v1/assets',
    headers={
        'Authorization': f'Basic {credentials}',
        'Content-Type': 'application/json',
    }
)

print(response.json())
```

</CodeGroup>

## Required Headers

Use the following headers on every request, depending on the type of operation:

<ParamField header="Authorization" type="string" required>
  Your Base64-encoded credentials in the form `Basic <base64(API_KEY:API_SECRET)>`. Required on every request.
</ParamField>

<ParamField header="Content-Type" type="string">
  Required on `POST` and `PUT` requests. Use `application/json` for JSON payloads and `multipart/form-data` for file uploads (e.g., KYC document uploads).
</ParamField>

<ParamField header="Idempotency-Key" type="string">
  Required on deposit and withdrawal requests. Supply a unique string per logical operation so that retried requests return the original response instead of creating a duplicate transfer. See [Idempotency](/platform/idempotency) for details.
</ParamField>

## Credentials by Environment

You receive separate credentials for the sandbox and production environments. Never use production credentials in sandbox, or vice versa.

| Environment | Base URL | How to obtain |
|---|---|---|
| **Sandbox** | `https://test-service.bluumfinance.com/v1` | Dashboard → Settings → API Keys |
| **Production** | `https://service.bluumfinance.com/v1` | Issued after compliance approval |

<Warning>
  Your API Secret is displayed **only once** at the time of creation. Copy it immediately and store it in a secret manager. If you lose it, you must generate a new key pair — Buildmarkets cannot recover a lost secret.
</Warning>

## Security Best Practices

Follow these guidelines to keep your credentials secure:

- **Use a secret manager** — Store credentials in AWS Secrets Manager, HashiCorp Vault, or Doppler. Never hard-code them in source files or commit them to version control.
- **Rotate keys every 90 days** — Regularly cycling credentials limits the blast radius of a compromised key.
- **Scope access to service accounts** — Assign credentials to dedicated service accounts rather than individual developer accounts.
- **Use environment variables** — Read `BLUUM_API_KEY` and `BLUUM_API_SECRET` from the environment to switch between sandbox and production without code changes.
- **Never log credentials** — Ensure your logging pipeline strips the `Authorization` header from request logs.

## Common Authentication Errors

If a request returns an HTTP `401`, check the error code in the response body:

| Error Code | Meaning | Resolution |
|---|---|---|
| `BLUM-401-001` | Missing `Authorization` header | Include `Authorization: Basic <credentials>` on every request |
| `BLUM-401-002` | Malformed credentials | Verify that you Base64-encoded the full `API_KEY:API_SECRET` string |
| `BLUM-401-003` | Invalid API key or secret | Confirm credentials are correct, active, and match the target environment |
| `BLUM-401-004` | API key inactive | Contact Buildmarkets support to reactivate your key |

<Tip>
  If you receive `BLUM-401-002`, the most common cause is encoding only the key or only the secret — make sure you encode the **combined** `API_KEY:API_SECRET` string, including the colon.
</Tip>
